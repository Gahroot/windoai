# Vertical Site Template

AI-powered receptionist website for a specific industry vertical. Next.js marketing site with blog, best-for pages, alternatives pages, and AI agent embed widget.

All brand-specific configuration comes from `src/lib/vertical.config.ts` — generated by the verticals hub scaffold script. Do NOT hardcode brand names, domains, or URLs.

## Project Structure

```
src/
├── app/                        # Next.js App Router pages
│   ├── layout.tsx              # Root layout (reads from vertical.config)
│   ├── page.tsx                # Home/landing page
│   ├── sitemap.ts              # XML sitemap generation
│   ├── api/                    # API routes (indexnow)
│   ├── blog/                   # Blog listing and [slug] routes
│   ├── book-demo/              # Demo booking page
│   ├── pricing/                # Pricing page
│   ├── alternatives/[slug]/    # Alternative solution pages
│   ├── best-for/[slug]/        # Industry-specific pages
│   ├── solutions/[slug]/       # Solution category pages
│   ├── faq/                    # FAQ page
│   ├── privacy/ & terms/       # Legal pages
│   └── [favicon/OG images]
├── components/
│   ├── ui/                     # Base UI components (shadcn/ui + Radix)
│   ├── sections/               # Landing page sections
│   │   ├── landing/            # Core landing sections
│   │   ├── pricing/            # Pricing-specific sections
│   │   ├── alternatives/       # Alternative product sections
│   │   └── best-for/           # Best-for sections
│   ├── layout/                 # Navbar, footer (reads from vertical.config)
│   ├── embed/                  # AI agent embed widget
│   ├── booking/                # Cal.com embed components
│   ├── effects/                # Visual effects (spotlight, etc.)
│   └── seo/                    # Structured data (reads from vertical.config)
└── lib/
    ├── vertical.config.ts      # SINGLE SOURCE OF TRUTH for brand config
    ├── crosslinks.ts           # Auto-generated cross-links to other verticals
    ├── nav-data.ts             # Navigation structure
    ├── alternatives/           # Alternatives page data (types + index)
    ├── best-for/               # Industry page data (types + index)
    ├── solutions/              # Solution category data (types + index)
    ├── content-factory/        # Programmatic content generation
    │   ├── constants/          # Reusable CTA, hero, feature constants
    │   └── factories/          # Alternative, best-for page generators
    └── validations/            # Zod form validation schemas

content/blog/                   # MDX blog posts (Fumadocs)
```

## Tech Stack

- **Framework**: Next.js 16 with App Router, React 19
- **Styling**: Tailwind CSS v4
- **UI**: shadcn/ui components with Radix UI primitives
- **Content**: Fumadocs MDX for blog
- **Animation**: Framer Motion
- **Icons**: Lucide React
- **Forms**: React Hook Form + Zod
- **SEO**: schema-dts for JSON-LD, IndexNow
- **Tracking**: Meta Pixel (conditional on config)
